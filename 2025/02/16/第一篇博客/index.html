第一篇博客

案例研究撰写

通过研究客户案例，可以突出产品是如何帮助客户在业务上取得成功的，同时又有助于吸引新客户

:::success

案例研究的结构框架“四段论”

flowchart LR

A(前言) --> B(挑战)

B --> C(方案)

C --> D(结果)

:::

一、前言介绍

介绍客户，突出客户的亮点

  在前言部分我们需要去介绍客户，突出客户的亮点，它的不同之处。这样在让客户为自己做背书的同时，也帮助客户完成了一定的曝光，达成==双赢==的局面。

二、面临挑战

聚焦到细节，以小见大

  在撰写客户案例之前，我们需要先对客户进行采访，这样会更利于我们发现客户所面对的挑战。客户面临的==挑战、关键问题或痛点==可以从以下几个方面展开：

  1. 这个问题是如何产生的？？
  2. 这是行业内普遍存在的问题还是客户遇到的新问题？？
  3. 这个问题是由于市场的变化或者其他不可预见的情况造成的吗？

三、解决方案

与客户面临的挑战一一对应

  这一部分的内容我们可以和第二部分的内容==一一对应==，那么，具体的解决方案可以从以下几个方面进行阐述：

  1. 这个解决方案是同行业通用的，还是针对该客户的情况而提出的具有个性化风格的方案？
  2. 在整个方案具体实施的过程中，有哪些团队或人员参与了，他们又分别做出了怎样的贡献？
  3. 最终的解决方案是什么？
  4. 客户如何通过产品或服务来解决问题？

四、结果呈现

不能罗列数字，要进行数据包装

  结果部分可以利用可视化的图表，说明客户利用自己提供的产品或服务获得了怎样的成果。例如，完成了多少目标，解决了多少问题。另外，对于罗列具体的数字我们可以进行一定的==包装==，方便引起用户的兴趣。
